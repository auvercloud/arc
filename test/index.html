optional parameters and default values

If the API call is OK, a temporary account is created and a customizable email including a confirmation link is sent to the email address.

The callback parameter contains the URL targeted by the confirmation link. By default, this parameter is set to API front end URL. It is recommended to provide your own URL with specific UI even if finally the confirmation parameters must be sent to API server.

If the account is confirmed by clicking the confirmation link included in the email, a second email containing a random-built password is then sent. If the account is not confirmed within a day, the temporary account is deleted.

See below the confirmation and conclusion default texts. To customize them, provide your own content using the optional parameters. See this page code source for a sample of API call and take a look at these best practices to design your own templates.

In the following examples of message,

  • %%business%% will be replaced by your business full name as defined in your account,
  • %%contact%% will be replaced by your contact email address as defined in your account,
  • %%link%% will be replaced by a confirmation link built with callback,
  • %%password%% will be replaced by a password generated by the server.
